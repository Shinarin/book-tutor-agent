# Book Tutor — AI 辅助读书系统设计

## 概述

一个由两个 Claude Code skill 组成的 AI 辅助读书系统。用户提供一本 Markdown 格式的书，系统通过 subagent 编排，自动生成全书总结和逐章教案，帮助用户以数倍效率完成学习。

## 问题与动机

读一本书需要从头到尾逐字阅读，耗时且低效。AI 可以作为导师，将原书内容以清晰易懂的方式重新组织和传授。但一次对话的 context window 无法容纳整本书，因此需要通过 subagent 编排分章处理，用文件系统作为中间状态传递媒介。

## 输入假设

- 一个完整的 Markdown 或 TXT 文件
- 章节用标题分隔（`# 第一章`、`## Chapter 1` 等）
- 文件大小不限，但单章节需能放入一个 subagent 的 context

## 输出结构

```
output/
├── summary.md          # 全书总结（含全书概览 + 各章摘要）
└── chapters/
    ├── chapter-01.md   # 第1章教案
    ├── chapter-02.md   # 第2章教案
    └── ...
```

## 架构：两个 Skill，两阶段流水线

### Skill 1: `book-summarize`

**触发**：用户输入 `/book-summarize`
**输入**：书的文件路径
**输出**：`output/summary.md`

#### 流程（串行）

1. **解析章节**：读取 Markdown 文件，按一级或二级标题（`#` 或 `##`）切分为章节列表
2. **逐章总结**（串行 subagent 链）：
   - 第1章 subagent：读取第1章原文 → 输出第1章总结，写入 `summary.md`
   - 第2章 subagent：读取「已有总结（第1章）」+ 第2章原文 → 输出第2章总结，追加到 `summary.md`
   - 第N章 subagent：读取「已有总结（第1~N-1章）」+ 第N章原文 → 输出第N章总结，追加到 `summary.md`
3. **生成全书概览**：最后一个 subagent 读取完整的 `summary.md`，生成一段全书概览，插入文件头部

#### 为什么串行？

每章的总结需要知道前面章节讲了什么，才能：
- 避免重复描述已总结过的概念
- 标注本章与前文的关联（"在第3章介绍了X的基础上，本章深入…"）
- 保持总结的递进性和连贯性

#### 章节总结 subagent prompt 要点

- 角色：你是一位学术导师，正在为学生准备读书笔记
- 输入：前面章节的累积总结（如有）+ 本章完整原文
- 任务：提炼本章核心概念、关键论点、重要细节
- 语言：中文，关键术语保留英文原文，格式如 `依赖注入（Dependency Injection）`
- 输出格式：
  ```
  ## 第N章：[章节标题]
  
  **核心主题**：一句话概括
  
  **关键内容**：
  - 要点1
  - 要点2
  - ...
  
  **与前文的关联**：（如适用）
  ```

### Skill 2: `book-teach`

**触发**：用户输入 `/book-teach`
**输入**：书的文件路径 + `summary.md` 路径
**输出**：`output/chapters/` 下每章一个教案文件

#### 流程（并行）

1. **解析章节**：同 Skill 1，切分章节列表
2. **并行生成教案**：为每章启动一个独立 subagent，每个 subagent 读取：
   - `summary.md`（全书总结，提供宏观上下文）
   - 本章原文
3. 各 subagent 独立输出教案文件到 `output/chapters/chapter-NN.md`

#### 为什么并行？

- 教案文字量可能与原文相当甚至更多
- 如果串行传递所有前序教案，context 会溢出
- 每个 subagent 只需全书总结（相对精简）+ 本章原文，context 负担可控

#### 教案 subagent prompt 要点

- 角色：你是一位经验丰富的导师，擅长将复杂知识以深入浅出的方式传授给学生
- 输入：全书总结 + 本章完整原文
- 任务：生成一份教案，结构如下：
  1. **导读**（~100字）：本章在全书中的位置，学完本章你将理解什么
  2. **深入浅出讲解**：
     - 将本章知识拆解为若干个知识模块
     - 每个模块用通俗的语言讲解，辅以类比和例子
     - 对复杂概念，先给出直觉理解，再展开细节
     - 关键术语保留英文原文
  3. **核心总结**：
     - 高密度要点提炼，适合快速回顾
     - 关键概念清单（中英对照）
     - 本章最重要的3-5个 takeaway
- 语言：中文授课，关键术语保留英文原文
- 风格：像一位好老师在课堂上讲课，不是在写教科书

## 章节解析策略

用正则匹配标题行来切分章节：

- 匹配模式：`^#{1,2}\s+.+`（一级或二级标题）
- 每个标题到下一个同级或更高级标题之间的内容为一个章节
- 章节编号从解析顺序生成（`chapter-01`、`chapter-02`…）
- 如果文件开头有标题前的内容（如前言），作为 `chapter-00` 处理

解析逻辑直接在 SKILL.md 的指令中用自然语言描述，由 Claude 在运行时执行（读取文件后按标题切分），不需要写脚本。

## 语言策略

- 总结和教案均使用中文
- 英文术语保留原文，格式：`中文翻译（English Term）`
- 如果原书是中文，术语部分不需要额外标注英文

## Context 预算分析

- **总结阶段**：累积总结随章节增长，但总结本身是压缩后的信息，通常远小于原文。一本20章的书，累积总结约 5000-10000 tokens，加上一章原文，总 context 可控
- **教案阶段**：每个 subagent 约 = 全书总结 (5000-10000 tokens) + 一章原文 (数千~数万 tokens) + prompt，完全在 context 范围内

## 错误处理

- 如果某章内容过长超出 context，skill 指令中提示用户手动拆分该章
- 如果文件不存在或路径错误，提示用户检查路径
