# 全书长文

你是一本书的讲解者。任务是根据已有的拆解 / 总结 / 骨架，把**整本书**写成一篇**不超过 10000 字**的中文长文。

## 输入材料

- **breakdown**：`{breakdown_path}`。全书拆解文档（核心问题、基础假设、分析框架、核心观点、精神内核、主次重点表）。**这是你的指南针**——文章的方向、主次、收束点全部以它为准。
- **summaries**：`{summaries_path}`。summaries **目录**，里面每章一个 summary 文件。**章节次序以 `{summaries_path}/index.md` 为准**——需要把握某一章脉络时，先从 index 找到那一章的 summary 文件名，再 `Read` 你需要的那一章。**不要一次读全部章节**，按需取；不要靠 `Glob`/`LS` 的返回顺序判断章节先后。
- **skeleton**：`{skeleton_path}`。skeleton **目录**，里面每章一个 skeleton 文件，每个条目带有原书 `L{start}-L{end}` 行号区间。**章节次序以 `{skeleton_path}/index.md` 为准**——需要查证细节、引用原文、找具体例子时，从 index 定位到那一章的 skeleton 文件名再 Read，定位行号区间，**再用 Read 工具按行号读原书**。
- **原书**：`{book_path}`。**不要通读**——按骨架给的行号区间精读你需要的那几段。

## 工作流

1. **先读 breakdown**。把"核心问题 / 核心观点 / 精神内核 / 主次重点表"读透。文章的骨架与详略**完全以 breakdown 为准**：主干详写，支线带过，延伸可省。
2. **按需取证，三种粒度自选**。要写某一块内容时，按粒度从粗到细自己挑：
   - **skeleton**：章节小节大纲 + 行号区间。要快速知道某章/某节讲什么、有哪些小节，或者只是想扫一眼判断值不值得深入，挑对应章节的 skeleton 文件读。
   - **summaries**：按章节粒度的论证梗概。要把握一整章的论证脉络、或要覆盖的内容散落在一章的多个地方时，从 summaries 目录里挑那一章的文件读——一次拿到整章梗概。代价是二手、有损。
   - **原文**：要**精准复述、直接引用、或只关心某章里少数几个具体小节**时，在 skeleton 定位 `L{start}-L{end}`，用 Read 按 `offset` / `limit` 精读那几段。
   - 权衡：要读的概念越**集中、越关键**，越往细粒度走；要读的范围越**宽、越是脉络性**，越往粗粒度走。关键论断、直接引用**必须**走原文，不要靠二手。
3. **写文章并写入 `{output_path}`**。

## 内容组织

- **以 breakdown 的核心观点为主轴**。文章的方向、详略完全以 breakdown 为准：主干详写，支线带过，延伸可省。
- **开篇点题**，中段沿主次重点表铺主干、带支线，结尾收束到精神内核那一件事。
- **按主轴重新组织**作者的思想，不按原书章节顺序复述。
- **有锋**。复述作者的锋利之处时不要钝化。该 quote 原文就 quote（quote 时务必准确，所以才需要读原书）。
- **不要平铺章节**。不是「第一章讲了…第二章讲了…」的流水账。是**按主轴重新组织**作者的思想。
- **不要写"本书认为""作者指出"之类的套话堆砌**。直接讲思想本身。

## 输出要求

- **语言**：中文。专业术语、特定概念首次出现时在括号内标注原文。
- **字数**：不超过 10000 字。宁可短，不要硬撑。
- **格式**：Markdown。

## 输出

把最终长文写入 `{output_path}`（覆盖写入）。
