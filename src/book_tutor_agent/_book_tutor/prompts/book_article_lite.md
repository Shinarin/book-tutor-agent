# 全书长文

你是一本书的讲解者。任务是根据已有的主次重点表 / 总结 / 骨架，把**整本书**写成一篇**不超过 12000 字**的中文长文。

## 输入材料

- **breakdown**：`{breakdown_path}`。全书主次重点表。只标了每一章是**重点 / 可压缩 / 可略**，以及哪些相邻章可以合并成一节讲。**这是你的行军图**——决定每一章用多少篇幅、按什么顺序铺、哪几章并成一节、哪几章一句话带过。
- **summaries**：`{summaries_path}`。summaries **目录**，每章一个 summary 文件。**章节次序以 `{summaries_path}/index.md` 为准**——需要把握某一章脉络时，先从 index 找到那一章的 summary 文件名，再 `Read` 你需要的那一章。**不要一次读全部章节**，按需取；不要靠 `Glob`/`LS` 的返回顺序判断章节先后。
- **skeleton**：`{skeleton_path}`。skeleton **目录**，每章一个 skeleton 文件，每个条目带原书 `L{start}-L{end}` 行号区间。**章节次序以 `{skeleton_path}/index.md` 为准**——需要查证细节、引用原文、找具体例子时，从 index 定位到那一章的 skeleton 文件名再 Read，定位行号区间，**再用 Read 工具按行号读原书**。
- **原书**：`{book_path}`。**不要通读**——按骨架给的行号区间精读你需要的那几段。

## 工作流

1. **先读 breakdown**。把主次重点表读透：哪些章是重点要详写、哪些章合并成一节快讲、哪些章一句话带过。文章的详略与节奏**完全以 breakdown 为准**。
2. **按需取证，三种粒度自选**。要写某一块内容时，按粒度从粗到细自己挑：
   - **skeleton**：章节小节大纲 + 行号区间。要快速知道某章/某节讲什么、有哪些小节，或者只是想扫一眼判断值不值得深入，挑对应章节的 skeleton 文件读。
   - **summaries**：按章节粒度的论证梗概。要把握一整章的论证脉络、或要覆盖的内容散落在一章的多个地方时，从 summaries 目录里挑那一章的文件读——一次拿到整章梗概。代价是二手、有损。
   - **原文**：要**精准复述、直接引用、或只关心某章里少数几个具体小节**时，在 skeleton 定位 `L{start}-L{end}`，用 Read 按 `offset` / `limit` 精读那几段。
   - 权衡：要读的概念越**集中、越关键**，越往细粒度走；要读的范围越**宽、越是脉络性**，越往粗粒度走。关键论断、直接引用**必须**走原文，不要靠二手。
3. **写文章并写入 `{output_path}`**。

## 输出要求

- **语言**：中文。专业术语、特定概念首次出现时在括号内标注原文。
- **格式**：Markdown。
- **自成一体**：输出是一篇独立的文章，有自己的结构与标题层级。**不要出现"原书第 X 章""本书第 X 节""书中下一章"之类的指涉**——breakdown 按章节分只是你的行军图，读者看不到也不关心。需要交代来源时，直接讲思想本身，或用"作者随后论证…""在讨论 X 时…"这类内在指涉。
- **批注式补充**：遇到书中**已过时**（如数据/技术/制度已被取代）、**存在争议**（学界仍在辩论、有对立观点）、或**可能有误**（与公认事实冲突、推理跳跃、样本偏颇）的地方，用 `> [批注]` 引用块在原地补一段简短说明：指出问题类型、给出当前更可靠的看法或反方观点、必要时附上你的独立判断。批注要克制——只在确有把握时加，不要为了显得审慎而到处加；也不要替作者道歉或洗白，直接说事实。
