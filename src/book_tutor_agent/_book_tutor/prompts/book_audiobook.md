读 `{teach_path}`，把它整篇改写为**适合朗读、用耳朵听**的有声书文稿，写入新文件 `{output_path}`。

## 目标

原文是一份"可以读"的教案，里面有 Markdown 标题、加粗、列表、代码块、公式、图片引用。听众听不到这些视觉结构——他们只能听到声音。你的任务是把这份教案改写成**一个人坐在你旁边、用嘴讲给你听**的样子。

## 改写要求

### 去掉所有视觉化的 Markdown 元素

- **图片引用**：整行删除。如果图片下方原本有 OCR 文字描述或图注，把这段文字自然融入到上下文的讲解里，不要保留"如图所示""见下图"这种依赖视觉的措辞。
- **标题符号**：删掉 `#` 号本身。**保留**标题文字作为分段提示，但要让它读起来像口语过渡，而不是像念标题。例如把 `## 三、信号与噪声` 改成自然的过渡句，比如"接下来我们聊聊信号和噪声"，或者"第三部分，信号和噪声"。
- **加粗 / 斜体**：去掉符号。如果原本靠加粗强调，改用口语化的强调方式——比如"注意，这里是关键"、"我特别想强调一点"、"记住这个词"，或者干脆把句子改写成结构上自然落在重音上的样子。不要每个加粗都套同一句话。
- **列表**：删掉符号，**改写成连贯的段落**。可以用"第一……第二……第三……"、"一方面……另一方面……"、"先说 A，再说 B，最后是 C"等方式把列表串起来。**不要**简单地把符号换成换行就完事。
- **数学公式**：改成自然语言朗读。例如：
  - `$E = mc^2$` → "能量等于质量乘以光速的平方"
  - `$\sum_{i=1}^{n} x_i$` → "把 x 从 i 等于 1 加到 n"
  - 复杂公式不要硬念，改成讲它的**含义和直觉**："这个式子说的是……"
- **数字**：保留阿拉伯数字原样，**不要**改写成中文数字。
- **表格**：改写成口语描述，按行或按列把内容讲出来。
- **链接**：保留 text，删掉 url。
- **引用块**：去掉 `>`，用"原文是这么说的——"、"作者写道"之类的口语方式自然引入。
- **人名**：原文是英文就保持英文，不要音译成中文。机构名、专有术语同理。

### 口语化改写

教案是写给眼睛看的，文字密度高、句子长、信息嵌套深。朗读时这种密度让听众跟不上。要做的事：

- **拆长句**。一句话超过 30 个字，考虑拆成两句。听众的工作记忆比读者短。
- **加过渡词和回指**。"前面说到……"、"还记得刚才那个例子吗"、"我们再回到一开始的问题"——帮听众跟上脉络。
- **省略掉视觉提示词**。"如下所示"、"上文提到"、"以下几点"、"右图"——改成"接下来"、"刚才说的"、"有这么几点"。
- **去掉"导师 / 老师"口吻**。原教案偶尔会出现"作为你的导师"这类师生关系措辞，改成朋友之间聊天的语气——是一个读过这本书的朋友跟你聊天，不是老师在给你上课。

### 保持内容完整

- **不要漏内容**。教案里讲的每个知识点、每个例子、每段批判性分析，都要在听感稿里有对应位置。可以重新组织顺序、可以合并、可以拆分，但不要"因为这段不好念就跳过"。
- **不要新增内容**。不要自己往里加教案没有的观点、例子、扩展。

## 输出

把改写后的完整稿件写入 `{output_path}`。输出本身仍然是 `.md` 文件，但里面**几乎没有 Markdown 语法**——就是一段段连续的中文，最多用空行分段。不要保留"改写说明""修订痕迹"。
