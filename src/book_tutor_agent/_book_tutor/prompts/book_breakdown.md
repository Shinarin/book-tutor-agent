# 拆解

输入一本书，输出它的框架。

## 这份产物给谁、做什么用

这不是给人读的读后感，也不是章节摘要。是给**讲解者**看的——他会拿这份文档作为讲解整本书的大方向：

- 决定**先讲什么、后讲什么、什么是主轴、什么可略**
- 不至于从第一页平铺到最后一页、没主次、把作者随手的例子和核心论断当成同等重量讲

所以这份文档的核心使命是：**给讲解者一张「主次重点图」**。

## 输入材料

- **全书核心知识点**：`{keypoints_file}`。
- **skeleton**：`{skeleton_path}`。原书的索引——每章一个 skeleton 文件，每个条目带原书 `L{start}-L{end}` 行号区间。**先读 `{skeleton_path}/index.md` 拿到章节次序**，再按需挑具体 skeleton 文件读；不要靠 `Glob`/`LS` 的返回顺序判断章节先后。
- **原书**：`{book_path}`。**不要通读**。要直接引用作者原话、或核对某个具体论断时再读，先用 skeleton 拿到行号区间，再用 `Read` 按 `offset`/`limit` 精读那几段。

## 怎么拆

### 1. 核心问题与时代语境

作者在答什么问题？为什么要写这本书？他是在对谁说话、在反对谁？

题目是 topic，问题是 question，动机是 motive。**只写书里和前言里能找到依据的**。

怎么找：读导言/前言，找"我写这本书是为了……"那一句。读不到，看作者反复回到的那一处——他绕了几百页都甩不掉的东西，就是动机。

动机可以是一场学术争论、一个时代现象、一个长期的智识困惑，也可以仅仅是把一个领域整理清楚。找不到明确线索时就直接留白，不要编。

**对讲解者的意义**：知道作者在答什么问题 = 讲解时知道主轴朝向哪里，不会把作者顺手的例子讲成中心议题。

**写完自检**：这一步写下的内容拿去描述同领域另一本书还成立吗？成立 → 太抽象，重写到具体。

### 2. 基础假设

作者立在什么不证之物上？

每本书都有几条天花板——作者不去证它们，他靠它们站着。动摇任何一条，整本书就塌。

操作问句两条，对着书问：
- 这书里什么是不许问的？（一问就要被作者归为外行）
- 什么是作者要相信才能往下写的？（少了它，后面所有论证失支点）

答出来的几条，就是这本书的基础假设。

亮出来不展开，证不证不在拆解者手里。

**对讲解者的意义**：用户若挑战这几条假设，讲解者应当意识到——这是在挑战整本书的地基，不是在挑战书里某个论点。讲解时遇到这些假设，应当**标注它们是假设、不是结论**，避免讲解者把作者的前提当作作者的论证成果来讲。

**写完自检**：这几条假设里**任意一条**被推翻，整本书还成立吗？还成立 → 那不是基础假设，是论点，重抽。

### 3. 分析框架

作者用什么看世界？

每个作者都有一台取景框。Darwin 的取景框是"变异 + 选择 + 时间"——他看蝴蝶看人看部落都是这台框。Marx 的取景框是"生产关系决定上层建筑"——他看历史看社会看艺术都是这台框。Freud 的取景框是"意识冰山一角，水下才是真主角"——他看梦看口误看政治都是这台框。

找作者的框：他反复用什么工具？他从什么角度切入？他看到 X 第一反应想到的是 Y——这个 Y 就是他的框。

框不一定是名词。可以是一个动作（"还原到 root cause"）、一个区分（"自然态 vs 文化态"）、一个对照（"显性逻辑 vs 隐性结构"）。

找框要够狠——一句话讲清楚，不是一个章节。

**对讲解者的意义**：取景框是讲解时的"翻译器"——遇到作者的任何具体例子，都可以用它解释"作者为什么这样看"。讲解者抓住框，听众就能跟着作者的眼睛看世界，而不只是听一堆零散结论。

### 4. 核心观点 / 结论

作者最后要读者带走什么？

不是"他举了多少例子"，不是"他论证了多少环节"。是：合上书，作者要读者带走的那个东西。往往就一两句话。

检验：把这句话单独抽出来，脱离作者上下文，它还有没有力量？如果只是复述书名，那不是核心观点——是标签。如果只是套话（"事物都是辩证的""万物相联""要独立思考"），那是空话——独立思考谁会反对？要抽就抽到"作者眼里的独立思考具体是什么、不是什么"这一层。

核心观点要锋利——能立得住、能被反驳、能被引用。"作者认为人性是复杂的"就立不住：它没有对立面，谁都没法反对。

最终落笔时，**能直接引用作者原话最好，引用不到用最贴近的复述**。

**默认只抽一条核心观点**。但有些书确实是多轴的——作者并行推进两三条独立论断，每条都立得住、贯穿全书、彼此不能互相还原。这种情况下可以列 2-3 条，按重要性排序，每条用作者本人的话或一句复述当小标题。判定标准很严：

- 拿掉其中任意一条，剩下的还能独立支撑半本书 → 是真·多轴。
- 一条是另一条的推论、应用、或具体化 → 不是多轴，是主轴 + 支线，回到单轴写法。
- 只是因为书厚、章节多、话题杂就想拆多条 → 不是多轴，是没抽到位，重抽。宁可单轴抽得狠，不要多轴凑数。

**对讲解者的意义**：这一条（或几条）是讲解的**主轴**——第 5 步的主次重点排序就以它为锚点。多轴时，第 5 步要在每条主轴下面接它自己的主次表。

### 5. 主次重点表

把原书的主要观点按"离主轴多远"排一遍。主轴就是第 4 步抽出来的那句核心观点。讲解者拿到这张表，就知道讲解时该按什么顺序铺、什么可略——任何章节、任何例子，最终都要能回挂到主轴上。

分三档：

- **主干**——直接支撑核心观点的几条。少了它们核心观点立不住。
- **支线**——作者展开、补充、应对反驳用的。拿掉不影响主轴，但能让主轴更丰满。
- **延伸**——作者顺手推到别处的应用、引申、案例。讲解时可略。

列**足够覆盖主轴所需**的主要观点 / 章节主题，每条标明属于哪一档，一句话说为什么放这一档（它跟核心观点是什么关系——支撑？补充？应用？）。

数量由主轴需要多少支撑决定，不是由书的厚度决定。主干列到"再去掉一条，核心观点就立不住"为止；支线/延伸列到"再加一条，对讲解者已无增量"为止。

**如果第 4 步是多轴的**：在第 5 步里为每条主轴各写一段主次表，用那条主轴本身的话当小标题领起，下面照常分主干/支线/延伸。同一个观点可能在第一条主轴下是主干、在另一条下是延伸——这正是多轴拆分的价值，能让讲解者看到同一内容在不同主轴下的权重差异。

**这是给讲解者的「主次重点表」**：讲解时按主干 → 支线 → 延伸的顺序铺，不要平铺。多轴时，讲解者还要决定主轴之间的讲解次序——通常按第 4 步排出的重要性次序，或按作者书中的推进顺序。

### 6. 精神内核 — 带走一件

合上书 10 年后只能记一件事——是哪一件？

不是把前面五步缩短。那是总结，是重复。这一件是从整本书萃出的内核：单独立住、可携带。

形态四选一。挑作者最依赖的那一种，不堆，不并列。注意：第 3 步已经把作者的分析框架（取景框）抽出来了，这里要么换一种形态（模型/洞见/概念/金句），要么把第 3 步的框架再萃一层、压成更可携带的形式——不要原样复读：

- **一个模型**——一个结构性认知工具（Kahneman：System 1/2；Marx：经济基础/上层建筑）
- **一个洞见**——一句被作者撬开的真相（"组织的命运不是被战略决定，是被结构决定"）
- **一个概念**——作者铸的新词（Taleb：反脆弱；KK：technium）
- **一个金句**——作者亲笔写出、可直接引用

交付：形态标签 + 内容本体 + 一句话点为什么是这一件。简短，别堆。

**多轴的书也只带走一件**——这条规则不放松。多轴是为了讲解时给出清晰的主次结构，不是给"内核"开口子。多轴时的处理方式有两种：

- 几条主轴背后其实有一个更上位的母概念（例如《人类简史》三场革命背后的"虚构能力塑造一切"），就抽那个母概念。
- 几条主轴确实并列、无法收敛到一，就**挑最锋利、最可携带的那一条**。

挑完三检：
- 合上书 10 年只能记一件，就是这件吗？
- 跟前面五步重了吗？重了且没有再萃一层 = 总结不是内核，重挑。
- 一个没读过的人听了这一件会想去翻原书吗？

**对讲解者的意义**：这是讲解的**收束点**。讲到全书末尾、或被问到"这本书到底想说什么"时，回到这一件。
